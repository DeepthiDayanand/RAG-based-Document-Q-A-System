Retrieval-augmented generation, or RAG, is an architecture for answering
questions with a large language model while grounding its answers in an
external document collection, rather than relying solely on what the model
memorized during training.

A RAG pipeline typically has two phases. In the ingestion phase, documents
are split into chunks, each chunk is embedded, and the embeddings are
stored in a vector database alongside the original text. In the query
phase, the user's question is embedded with the same model, the vector
database returns the most similar chunks, and those chunks are inserted
into the prompt sent to the language model.

Chunk size is an important tuning parameter. Chunks that are too large
dilute the relevance of a single passage and waste context window space;
chunks that are too small can lose surrounding context needed to answer a
question correctly. Overlapping consecutive chunks by a small amount helps
avoid losing information right at a chunk boundary.

Common ways to evaluate a RAG system include recall@k, which measures
whether the correct source document appears among the top k retrieved
chunks, and end-to-end answer accuracy, which checks whether the final
generated answer is actually correct, since good retrieval does not
guarantee a good final answer.
